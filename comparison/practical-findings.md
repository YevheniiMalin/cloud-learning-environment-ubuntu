# Practical Findings

## Overview
This document summarizes the most important practical findings from the project.

The goal is to highlight what was learned from deploying, configuring, and evaluating both MicroStack and Apache CloudStack in a real Ubuntu Server environment.

## Key Findings

### 1. Simplicity vs Capability
MicroStack offered a much simpler starting point and faster deployment experience.

Apache CloudStack required more setup effort, but provided broader and deeper platform management capabilities.

### 2. Educational Suitability
MicroStack is especially suitable for introductory learning environments where the main goal is to understand cloud basics and virtualization concepts.

Apache CloudStack is more suitable when the learning goal includes broader administration, infrastructure structure, and deeper service management.

### 3. Infrastructure Understanding
Working with both platforms improved understanding of:
- virtualization platforms
- system preparation
- service dependencies
- networking
- storage roles
- user and resource management

### 4. Multi-User Perspective
The project highlighted how important user isolation and role separation are in shared infrastructure environments.

A learning environment must not only work technically, but also support safe usage by multiple users.

### 5. Documentation Value
A major practical lesson from the project was the importance of clear documentation.

Technical work becomes more valuable when installation, configuration, and observations are documented in a way that others can understand and reuse.

## Career Relevance
This project is relevant for infrastructure-focused roles because it demonstrates:
- practical server environment work
- virtualization platform evaluation
- structured technical documentation
- platform comparison and analysis
- awareness of user isolation and service environment design

## Final Reflection
The project showed that technical environments should be evaluated not only by features, but also by usability, administration effort, and suitability for the intended purpose.

That perspective is important in real IT environments where technology decisions affect both users and operations.
